API


Name
Type
Default
Description
fields
Array
[]
List of table fields.
Each item in array, is object that can have
value:String, icon:String, text:String, class:String, width:[Number, String] and formatter:Function prop.
local
Boolean
false
Is it local table rows or should call api to fetch rows
ajax
---
null
Ajax provider object, if it's not local.
rows
Array
[]
List of table rows, just if it's local.
api
String
"/items?page:{page}&limit={limit}"
Api url, if it's not local.
{page} and {limit} part inside api, passed by fv-table automatically.
header
Boolean
true
Show table header.
footer
Boolean
true
Show table footer.
paginate
Boolean
true
Doe's not api response return all rows alltogether?
If yes, it means you can use paginate to let user navigate into pages.
api-rows-key
Enum [String, null]
null
Rows key from api response.
api-total-count-key
Enum [String, null]
null
If your api response return total-count of rows key, set key into this.
api-finished-key
Enum [String, null]
null
If your api response return finished boolean prop, set key into this.
Note that you can put "!" at start of string, if your api return not-finished boolean prop.
api-next-page-key
Enum [String, null]
null
If your api response return next page value prop, set key into this.
api-previous-page-key
Enum [String, null]
null
If your api response return previous page value prop, set key into this.
clickable-rows
Boolean
true
Let user click on rows.
limit
Number
15
fv-table set this to {limit} part of "api".
menu
Boolean
false
Allow menu button and event.
check-list
Boolean
false
Allow select multiple rows for menu event.
initial-state
Object
null
Initial state of table in api mode. You can find value by calling getState method.
